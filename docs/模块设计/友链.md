## 职责与边界

友链模块负责校验并展示 `data/friends.json`，提供中英文静态页面、站点外链、头像失败回退、导航入口、SEO 和响应式布局。Notion 数据查询、审核和图片下载仍由 Notion 同步模块负责；本模块不在网页请求期间访问 Notion，也不提供在线申请表单。

## 结构与数据流

| 路径                                 | 用途                                 |
| ------------------------------------ | ------------------------------------ |
| `lib/friends/schema.ts`              | 友链文件和单条友链的公共 Zod 规则    |
| `lib/friends/repository.ts`          | 构建时读取并校验 `data/friends.json` |
| `features/friends/friends-page.tsx`  | 页面标题、数量、列表和空状态         |
| `features/friends/friend-card.tsx`   | HeroUI 友链卡片与外部链接            |
| `features/friends/friend-avatar.tsx` | 头像加载失败后的首字母回退           |
| `features/friends/friends.css`       | 桌面、手机和明暗主题样式             |
| `app/(zh)/links/page.tsx`            | 中文静态路由与元数据                 |
| `app/(en)/en/links/page.tsx`         | 英文静态路由与元数据                 |
| `tests/unit/friends.test.ts`         | 文件结构和 URL 安全规则              |
| `tests/e2e/home.spec.ts`             | 页面、外链、回退和语言切换           |

同步脚本先下载审核通过的头像，再把站点名称、简介、HTTPS 地址和本地头像路径写入 JSON。Next.js 构建时通过 `getFriendLinks` 解析 JSON；字段不合法会直接让构建失败。页面只接收已经验证的数据，不带 Notion Token 或数据库信息。

## 关键决策

1. 页面使用 `/links/` 与 `/en/links/`，和站内其他栏目保持同一套语言前缀与尾斜杠规则。
2. 外部站点在新标签页打开，并设置 `rel="noreferrer"`；站内语言切换和导航继续留在当前标签页。
3. 头像路径只接受 `public/` 下的绝对站内路径。同步阶段把远程头像下载到本地，减少第三方站点故障和隐私泄露对页面的影响。
4. 浏览器仍可能遇到文件删除、损坏或缓存异常。`FriendAvatar` 监听图片错误，失败后显示站名首字母，卡片尺寸保持不变。
5. 友链卡片使用 HeroUI Card，整卡只有一个外部链接；没有嵌套链接，键盘焦点和鼠标悬停状态一致。

## 当前实现

中文和英文页面共用 `FriendsPage`。页面顶部沿用全站 `PageIntro`，列表在桌面使用两列、手机使用单列；卡片显示头像、站名、域名、两行简介和外链图标。当前迁入参考项目中的 Betsy Blog，后续真实 Notion 同步会用审核结果覆盖 JSON 和头像目录。

友链页已加入桌面导航、移动菜单、页脚和 sitemap。静态产物检查要求中英文页面都存在，部署冒烟会访问中文友链页。

## 验证方式

- `tests/unit/friends.test.ts` 验证安全 URL、本地头像路径和严格字段结构。
- `tests/unit/navigation.test.ts` 验证友链导航、激活状态和语言切换。
- `tests/unit/seo.test.ts` 验证中英文友链地址进入 sitemap。
- Playwright 拦截真实头像请求，确认首字母回退可见、外链属性正确、语言切换进入英文页面。
- 浏览器视觉检查覆盖桌面 `1440x1000` 和手机 `390x844`，检查裁切、换行、卡片高度和控制台错误。

## 待扩展项

- 当前没有在线友链申请接口。若后续需要恢复参考项目的申请表单，应单独设计 Cloudflare Pages Function 的字段校验、限流、防滥用和 Notion 写入权限。
- 页面不主动检查外部站点是否在线，失效链接由 Notion 审核流程处理。
