## 职责与边界

本模块负责三个广告位置、AdSense 脚本、自营广告内容和文章微信赞赏弹窗。广告点击直接进入配置的目标地址，赞赏资金由微信直接收取，本站不处理订单、金额、用户身份或支付回调。

第三方评论、访问统计和 AI 搜索继续由各自模块管理。广告或赞赏关闭、配置缺失以及第三方脚本失败都不能影响正文阅读。

## 结构与数据流

| 路径                                            | 职责                                               |
| ----------------------------------------------- | -------------------------------------------------- |
| `site.config.ts`                                | 广告位置、Provider、AdSense ID、广告内容和赞赏配置 |
| `lib/monetization/config.ts`                    | 校验公开配置并生成页面可用数据                     |
| `components/integrations/adsense-script.tsx`    | 只在存在启用的 AdSense 广告位时延后加载脚本        |
| `lib/seo/metadata.ts`                           | 启用广告时输出 Google AdSense 账户声明             |
| `public/ads.txt`                                | 声明 Google Publisher 与直接合作关系               |
| `features/monetization/ad-placement.tsx`        | 渲染 AdSense 单元或自营广告                        |
| `features/monetization/adsense-unit.tsx`        | 向 AdSense 队列注册当前广告单元                    |
| `features/monetization/sponsorship-section.tsx` | HeroUI 赞赏弹窗和二维码状态                        |
| `features/monetization/monetization.css`        | 三类广告位、赞赏区和弹窗的响应式样式               |
| `tests/unit/monetization-config.test.ts`        | 配置开关、格式和安全边界单测                       |
| `tests/e2e/home.spec.ts`                        | 赞赏入口、弹窗和评论顺序验证                       |

数据流如下：`site.config.ts` 先经过 `lib/monetization/config.ts` 校验，页面只接收整理后的广告或赞赏数据。启用 AdSense 后，公共布局加载一次 Google 脚本，各广告单元再注册自己的 slot。自营广告完全使用仓库配置，不执行外部 HTML 或脚本。

## 关键决策

1. 广告位置固定为 `home`、`posts` 和 `article`。首页位置处于作者区与近期文章之间；列表位置处于当前页第二篇文章之后并横跨两列；正文位置处于正文与前后篇导航之间。
2. 每个位置都有独立 `enabled` 和 `provider`。`provider` 可选 `adsense` 或 `custom`，同一站点可以混合使用两种来源。
3. 自营广告只接受标题、说明、链接和可选站内图片。链接只允许 HTTPS 或站内地址，图片只允许 `/images/`，不接受任意 HTML。
4. 没有启用的 AdSense 位置时，页面不输出 Google 脚本。广告根节点也不会渲染，因此不会留下空白区域。
5. 赞赏二维码只允许放在 `public/images/` 对应的站内路径。当前仓库没有真实收款码，入口保持可见并显示“暂未开放”；配置真实路径后自动显示二维码。
6. 赞赏使用 HeroUI Modal。弹窗关闭、键盘焦点和遮罩行为复用组件库，二维码使用固定正方形尺寸，减少布局跳动。
7. AdSense 在 hydration 后由统一客户端加载器异步插入，标签使用稳定 ID 防止站内导航重复加载；加载失败时移除标签，让后续挂载有机会重试。

## 当前实现

### 广告配置

`integrations.advertising.adsenseClientId` 保存公开 Publisher ID。三个位置分别配置：

- `enabled`：是否显示该位置。
- `provider`：使用 `adsense` 或 `custom`。
- `slotId`：AdSense 广告单元 ID。
- `custom`：自营广告的标题、说明、目标链接和可选图片。

当前文章位置已经启用，使用参考项目正在运行的 Publisher ID `ca-pub-1149581082118045` 和文章 Slot ID `6077231481`。首页、文章列表位置保持关闭；这两个位置需要在 AdSense 后台新建适合版式的单元后再打开，避免混淆位置数据。

只要存在启用的 AdSense 位置，公共页面就在 hydration 后异步加载一次 Google 脚本，并输出 `google-adsense-account` 元数据。`public/ads.txt` 使用同一 Publisher；构建检查账户声明和 CSP，Playwright 检查浏览器挂载的脚本没有 `inert`、模拟脚本已经执行且正文 slot 已进入 AdSense 队列。

### 赞赏配置

`integrations.sponsorship.enabled` 控制整段赞赏区，`qrCodeSrc` 保存站内二维码路径，`copy` 保存中英文标题、说明、按钮、弹窗和无二维码提示。当前赞赏区已开启，`qrCodeSrc` 为空，因此不会展示可扫码的收款信息。

## 验证方式

- `pnpm test:unit` 验证关闭状态、AdSense ID/slot、自营广告链接与图片、二维码路径。
- `pnpm test:e2e` 验证赞赏区位于评论前、HeroUI 弹窗可打开、无二维码状态清楚可见。
- `pnpm build` 验证静态导出、CSP 和 AdSense 脚本开关。
- 浏览器分别检查桌面和手机下广告高度、弹窗尺寸、二维码比例、暗色主题和页面溢出。
- AdSense 审核、实际填充率和广告内容由 Google 控制，生产发布后需要在浏览器与 AdSense 后台人工确认。
