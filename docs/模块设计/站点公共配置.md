# 站点公共配置

## 设计

### 职责

站点公共配置负责集中保存更换博客所有者时需要调整的公开信息，包括品牌、作者、头像、栏目介绍、社交链接以及第三方服务公开 ID。页面、SEO、RSS 和评论系统只读取这份配置，减少散落在组件里的硬编码。

### 目录结构

| 路径                               | 用途                                |
| ---------------------------------- | ----------------------------------- |
| `site.config.ts`                   | 唯一的公开配置编辑入口              |
| `lib/site-config.ts`               | 多语言读取助手和站点 URL 校验       |
| `lib/giscus/config.ts`             | 评论开关与 Giscus 字段格式校验      |
| `tests/unit/site-config.test.ts`   | 多语言资料、集成开关和站点 URL 单测 |
| `tests/unit/giscus-config.test.ts` | 评论配置启用、关闭和异常组合单测    |

### 关键决策

1. `site.config.ts` 放在仓库根目录，作为更换站点资料时首先编辑的文件。
2. 配置按品牌、作者、栏目文案、社交链接和第三方集成分组；中英文值使用同一套语言键。
3. 头像可以填写 `public/` 下的站内路径或公开 HTTPS URL，分享图可以填写站内路径。社交链接可以单独关闭，关闭后页面不会留下空入口。
4. Giscus、AdSense 客户端 ID、Cloudflare Web Analytics Token 都会暴露给浏览器，可以保存在公共配置中。
5. Notion Token、未来 AI 服务密钥等私密值只存放在环境变量中。`NEXT_PUBLIC_SITE_URL` 由部署环境提供，以便本地、预览和生产使用不同域名。
6. 第三方服务默认关闭。启用 Giscus 后会检查必需字段；AdSense 和访问统计在对应功能接入时沿用同样规则。

## 实现细节

### 配置分组

| 分组                  | 主要字段                                               |
| --------------------- | ------------------------------------------------------ |
| `brand`               | 站名、字标、品牌缩写、图书标识、简介、页脚短句、分享图 |
| `author`              | 作者名、签名、首页标题、关于页文案、头像与替代文字     |
| `sectionDescriptions` | 文章、归档和图书栏目的中英文说明                       |
| `socialLinks`         | 平台、显示名称、URL 和开关                             |
| `integrations`        | Giscus、AdSense、Cloudflare Web Analytics              |

`getLocalizedSiteConfig(locale)` 从多语言配置中取出当前页面需要的值。客户端组件只读取公开信息；服务端元数据和 RSS 也复用同一个结果。

外部头像通过浏览器直接请求，图片服务仍能看到访问者 IP 和 User-Agent。头像请求使用 `no-referrer`，不会附带当前博客页面地址；对隐私要求更高时应把头像放到 `public/images/`。

### 第三方服务

- `integrations.comments.enabled` 控制文章评论入口。开启后 `repo`、`repoId`、`category` 和 `categoryId` 必须完整。
- `integrations.adsense` 预留 `clientId` 和开关，当前不会加载 Google 脚本。
- `integrations.analytics` 预留 Cloudflare Web Analytics 的公开 Token 和开关，当前不会发出统计请求。
- Playwright 使用 `PLAYWRIGHT_TEST_COMMENTS=1` 注入固定测试评论配置，不读取真实仓库，也不会访问真实 GitHub 数据。

### 更换站点资料

1. 修改 `site.config.ts` 的品牌和作者资料。
2. 将头像与分享图放入 `public/`，更新对应站内路径。
3. 修改或关闭社交链接。
4. 按需启用评论、广告和访问统计；保持未使用服务关闭。
5. 修改部署环境中的 `NEXT_PUBLIC_SITE_URL`，再运行完整项目门禁。

## 测试方法

- 单测确认两种语言都能读取完整的站点名称、作者和描述。
- 单测确认尚未配置的第三方服务默认关闭。
- Giscus 单测覆盖关闭、完整启用、格式错误和 Playwright 测试配置。
- E2E 继续验证首页标题、作者区、文章评论和英文 RSS 路径。

## 当前限制

- AdSense 与访问统计只有配置结构，脚本加载和隐私说明将在各自功能任务中完成。
- 当前只支持一位站点作者。
