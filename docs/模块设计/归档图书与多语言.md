# 归档、图书与多语言

## 设计

### 职责

本模块负责把内容系统中的文章和图书组织成归档、标签详情、图书列表和图书详情页，并让中英文页面之间可以稳定切换。它处理构建期静态路由、日期和数量文案、图书进度与章节入口，以及内容译文的路径映射；搜索、RSS、站点地图和分享元数据仍由阶段 5 完成。

| `tests/e2e/home.spec.ts`              | 归档、标签、图书、章节和语言切换浏览器流程 |

### 关键决策

1. 归档使用开放时间线，年份和文章数量来自同一份已发布文章清单。标签数量也从这份清单计算，避免页面各自维护数字。
2. 标签页、图书页和图书详情都保留为 Server Component。内容查询在服务端完成，浏览器只接收渲染结果和语言路径表。
3. 图书进度使用 HeroUI `ProgressBar`。frontmatter 只接受 `0` 到 `100` 的整数，正文二级标题代表顶层章节，三级标题代表章内小节。
4. 文章和图书通过 `translationKey` 建立译文关系。语言切换优先进入同一内容的译文，缺少译文时回到目标语言首页。
5. 根布局并行读取文章和图书，并只向客户端传递“当前路径到目标路径”的轻量表，不传正文、目录和搜索文本。
6. 标签没有跨语言对应关系。标签详情切换语言时回到目标语言首页，避免把中文标签名称硬套到英文内容上。
7. Next.js 静态构建会把非 ASCII 动态参数交给页面时保留 URL 编码。标签路由入口统一解码一次，再执行内容查询；链接本身保留原始标签文本，由路由器和浏览器完成编码。
8. 语言切换查表时同时尝试浏览器原始路径和解码后的路径，保证从客户端进入中文 Unicode slug 后仍能找到英文译文。
9. 图书列表只展示二级章节入口。图书详情的长目录同样只保留二级章节，避免真实长书的数百个小节撑开卡片和侧栏。

## 实现细节

### 归档与标签

1. `ArchivePage` 读取当前语言的已发布文章，调用 `groupArticlesByYear` 和 `getTagSummaries` 得到时间线与主题统计。
2. 日期统一用 `Intl.DateTimeFormat` 和 UTC 格式化，避免构建机器时区改变显示日期。
3. `generateStaticParams` 为当前语言的每个标签生成详情路径，`dynamicParams = false` 保证静态部署不会接收未知标签。
4. 页面拿到 `[tag]` 参数后用 `decodeTagSegment` 还原一次，再查询文章。没有文章时进入站点 404。
5. 标签详情复用文章卡片，不再维护第二套摘要布局和跳转规则。

### 图书

1. 图书列表先排除草稿，再按 `order` 排序。每本图书显示状态、说明、进度、章节入口和正文入口。
2. `BookProgress` 使用 HeroUI 组合式进度条，输出可访问的 `progressbar`、数值和可视轨道。
3. 章节入口使用内容分析器生成的二级标题锚点。点击列表章节可进入图书详情并定位到对应标题，章内三级标题只在短目录中展示。
4. 图书详情路由在构建期列出 slug，找不到已发布图书时进入站点 404。
5. Next.js 16 会把空的动态参数列表当成未实现静态参数。某种语言没有图书时，该语言路由生成内部 `_empty` 参数并输出带 `noindex` 的缺失状态；这个地址不会写入导航、sitemap 或搜索索引。

### 语言切换

1. 根布局并行取得全部文章和图书，`buildContentLocaleRoutes` 为每个已发布内容生成目标语言路径。
2. 有译文时，路径表指向相同 `translationKey` 的目标内容；没有译文时指向目标语言首页。
3. `LocaleSwitcher` 根据当前 `usePathname` 查询路径表，并兼容 URL 编码与解码两种形式。普通列表页保留当前栏目，未知的文章、图书和标签动态页回目标语言首页。
4. 路径表只包含字符串，不包含文章或图书数据，避免把构建期正文带入客户端脚本。

### 测试方法

- 单测覆盖归档分组、标签统计、图书排序、进度范围、文章和图书译文映射、缺少译文回退及标签动态参数还原。
- E2E 覆盖归档年份与数量、进入中文标签详情、真实图书数量、章节定位、进度和无译文回退。
- `pnpm build` 验证全部标签和图书详情都能通过 `output: "export"` 生成静态文件。

## 当前限制

- 标签只按完全一致的名称聚合，没有别名、父子标签和跨语言映射。
- 图书进度由 frontmatter 手工维护，当前不会根据已写章节自动计算。
- 图书封面使用项目内的排版式封面，后续可在内容模型中增加本地封面图片字段。
