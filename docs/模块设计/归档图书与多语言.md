# 归档、图书与多语言

> **模块定位**：组织归档、标签、目录式图书、独立章节阅读和内容语言切换
>
> **对应代码**：`features/archives/`、`features/tags/`、`features/books/`、`app/{(zh),(en)}/**/books/`、`lib/content/locale-routes.ts`
>
> **所属 M 里程碑**：[M4：归档、图书与多语言](../roadmap.md#阶段-4归档图书与多语言)
>
> **当前状态**：已完成
>
> **最近更新时间**：2026-07-16

## 职责与边界

本模块把内容系统中的文章和图书组织成归档、标签详情、图书列表、图书目录和章节阅读页，并为中英文内容建立稳定的切换路径。搜索、RSS、通用 SEO 生成和生产部署分别由其他模块维护；本模块只向它们提供图书和章节路由数据。

## 结构与数据流

| 路径                                         | 用途                             |
| -------------------------------------------- | -------------------------------- |
| `features/archives/`                         | 年份归档、标签统计和时间线       |
| `features/tags/`                             | 标签路径和标签文章列表           |
| `features/books/book-card.tsx`               | 固定高度图书卡片和两章预览       |
| `features/books/book-detail-page.tsx`        | 图书介绍、元数据和完整章节目录   |
| `features/books/book-chapter-page.tsx`       | 单章正文、全书目录和上下章导航   |
| `app/{(zh),(en)}/**/books/[slug]/`           | 图书目录静态路由                 |
| `app/{(zh),(en)}/**/books/[slug]/[chapter]/` | 独立章节静态路由                 |
| `lib/content/locale-routes.ts`               | 文章、图书和章节的语言切换路径表 |
| `lib/seo/sitemap.ts`                         | 图书目录和章节 sitemap 条目      |

构建时 `loadBooks` 读取 `content/books/{zh,en}/<book>/index.md`，再读取同目录章节文件。查询层过滤草稿并按 `order` 排序。图书列表只拿章节数量和前两章；图书目录页拿完整章节清单；章节页只渲染当前章节正文，同时拿相邻章节生成导航。

## 关键决策

1. 图书是资料、教程和翻译内容的集合，不维护用户阅读状态，因此数据模型和界面都不再包含阅读进度。
2. 图书列表卡片在桌面和手机视口分别使用稳定高度。标题和简介限制行数，章节只预览前两项，章节多寡不会改变卡片高度。
3. `/books/[slug]/` 只展示图书介绍和完整目录；`/books/[slug]/[chapter]/` 每次只渲染当前章节，避免整本书进入一个 HTML 页面。
4. 桌面章节页使用左侧固定全书目录，当前章节有明确状态；手机使用可展开的紧凑目录，正文底部保留上一章、目录和下一章。
5. 图书、章节页面继续使用 Server Component。正文、章节清单和导航在构建期完成，浏览器无需再请求内容接口。
6. 图书和章节可分别使用 `translationKey` 关联译文。有章节译文时切到对应章节；只有图书译文时回到目标语言的图书目录；整本书没有译文时回目标语言首页。
7. 章节路由进入 canonical、hreflang 和 sitemap。旧站同 slug 章节地址直接复用；旧日语书名通过单跳 `301` 映射到新的书籍 slug，并保留章节参数。

## 当前实现

### 归档与标签

- 归档按发布时间分组，标签数量从同一份已发布文章清单计算。
- 标签详情复用文章卡片，中文动态参数在查询前安全解码。
- 标签没有跨语言关系，切换语言时回目标语言首页。

### 图书列表

- 图书先排除草稿，再按 `order` 排序。
- 每张卡片显示状态、作者或译者、两行简介、章节数、更新时间、最多两个章节入口、标签和目录入口。
- 卡片不显示阅读进度；桌面和手机卡片都由固定网格尺寸约束。

### 图书目录

- 顶部展示书名、状态、作者、译者、日期、标签和章节总数。
- `index.md` 正文作为图书介绍渲染，右侧或移动端下方列出完整章节目录。
- “开始阅读”进入第一篇已发布章节。

### 章节阅读

- 页面标题、正文和文内二三级标题只来自当前章节文件。
- 桌面左侧目录限制在视口高度内滚动，手机目录使用原生 `details` 保持无 JavaScript 可用。
- 底部导航从排序后的已发布章节计算，不会链接草稿。
- 中英文静态路由都提供空内容占位参数，满足 Next.js 静态导出对动态路由的要求。

## 验证方式

- `tests/unit/book-content.test.ts` 覆盖目录式读取、章节排序、草稿过滤、重复顺序和上下章导航。
- `tests/unit/content-schema.test.ts` 覆盖无进度的图书 schema 与章节顺序约束。
- `tests/unit/locale-routes.test.ts` 和 `tests/unit/seo.test.ts` 覆盖章节语言回退与 sitemap。
- `tests/e2e/home.spec.ts` 覆盖卡片等高、进度条移除、章节直达、单章正文、上下章切换和移动目录。
- `pnpm build` 检查 26 个章节静态页面、canonical、hreflang、sitemap 和全部站内章节链接。

## 待扩展项

- 当前图书使用排版式封面；以后可以在图书 frontmatter 增加本地封面图片。
- 当前三本书只有中文。加入英文图书时，需要为书籍和对应章节配置成对的 `translationKey`。
- 章节目录按一个连续列表显示；章节达到数十篇后，可增加卷或分组字段。
