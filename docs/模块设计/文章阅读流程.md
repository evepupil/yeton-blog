# 文章阅读流程

## 设计

### 职责

文章阅读流程负责把内容系统生成的数据呈现为首页近期文章、文章列表和独立正文页面，并完成标签筛选、分页、目录锚点和前后篇跳转。它不负责归档、标签详情页、搜索索引和完整 SEO，这些能力在后续阶段实现。

### 目录结构

| 路径                                  | 用途                                   |
| ------------------------------------- | -------------------------------------- |
| `features/home/`                      | 首页作者区、真实标签和近期文章组合     |
| `features/posts/posts-page.tsx`       | 文章列表的服务端数据入口               |
| `features/posts/article-browser.tsx`  | 标签筛选、分页和 URL 状态              |
| `features/posts/article-card.tsx`     | HeroUI 文章摘要卡片                    |
| `features/posts/article-page.tsx`     | 正文页头、封面、正文、目录和前后篇组合 |
| `features/posts/markdown-content.tsx` | 安全 Markdown 渲染和标题锚点           |
| `features/posts/article-toc.tsx`      | 桌面固定目录和手机目录块               |
| `features/posts/posts.css`            | 文章列表与正文的桌面、手机样式         |
| `app/posts/[slug]/`                   | 中文文章静态路由                       |
| `app/en/posts/[slug]/`                | 英文文章静态路由                       |
| `tests/e2e/home.spec.ts`              | 核心阅读流程浏览器测试                 |

### 关键决策

1. 首页、列表和正文都是 Server Component 的入口。只有需要读取浏览器 URL 和响应按钮操作的 `ArticleBrowser` 是 Client Component。
2. 客户端列表只接收 `ArticlePreview`，不包含正文、搜索纯文本、目录和源路径，减少页面脚本数据。
3. 标签筛选使用 HeroUI Button，文章摘要使用 Card 和 Chip，空结果与分页使用对应 HeroUI 组件。项目样式只调整尺寸、颜色和排版。
4. 筛选与页码保存在 `?tag=`、`?page=` 中。组件通过浏览器历史记录同步状态，支持刷新和后退，也不会触发额外的服务端数据请求。
5. 每种语言的 `[slug]` 都由 `generateStaticParams` 在构建时列出，`dynamicParams = false` 防止静态部署生成未知路径。
6. Markdown 使用 `react-markdown` 渲染，不开启原始 HTML。`rehype-slug` 与内容分析器共用 GitHub slug 规则，目录链接和正文标题保持一致。
7. 文章始终在当前标签页进入独立地址。刷新、复制链接和浏览器前进后退都能回到同一篇文章。

## 实现细节

### 首页与列表

1. `HomePage` 和 `PostsPage` 通过缓存后的 `getAllArticles` 读取文章。
2. 内容查询先排除草稿、筛选语言，再按置顶和日期排序。
3. 首页取前 3 篇文章和前 6 个标签；每个“阅读全文”直接指向对应的独立文章地址。
4. 列表把轻量摘要传给 `ArticleBrowser`。浏览器根据 URL 计算当前标签、页码和可见文章，不另存一份容易不同步的派生状态。
5. 用户选择标签或页码时写入浏览器历史并触发重新计算。无效标签回到全部，无效页码限制在实际页数内。

### 正文

1. 路由根据语言和 slug 查找已发布文章，找不到时进入站点 404。
2. 页头显示发布日期、字数、阅读时间、摘要和标签；标签回到带筛选参数的文章列表。
3. 正文由服务端生成 HTML。二、三级标题带稳定 id，目录链接滚动到对应位置。
4. 桌面目录固定在正文右侧，手机端移到正文上方，避免压缩阅读宽度。
5. 前后篇沿用文章列表顺序。置顶文章优先，其余文章按发布日期从新到旧排列。

### 测试与视觉检查

- 单测覆盖轻量摘要、文章查找、前后篇和动态文章语言切换回退。
- E2E 覆盖首页进入英文列表、首页进入真实正文、目录锚点、前后篇、标签筛选、分页恢复和移动菜单。
- 内置浏览器检查 `1440 x 1000` 与 `390 x 844`。首页、列表和正文没有横向溢出、资源缺失或控制台错误。

## 当前限制

- 文章详情的精确译文切换在阶段 4 接入 `translationKey` 后完成。
- 当前正文支持安全 Markdown 语法，不执行 MDX 中的任意 React 组件或原始 HTML。
- 标签详情地址、搜索结果摘要和分享元数据分别在阶段 4、阶段 5 实现。
