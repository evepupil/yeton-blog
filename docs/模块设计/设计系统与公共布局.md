# 设计系统与公共布局

## 设计

### 职责

本模块负责把 HTML 原型中的视觉规则、站点顶栏、导航、主题切换、语言切换、页脚和通用页面标题整理为可复用的 Next.js 组件。首页首屏使用统一站点配置和真实内容数据，公共布局不保存博客所有者的硬编码资料。

### 目录结构

| 路径                          | 用途                                   |
| ----------------------------- | -------------------------------------- |
| `app/globals.css`             | 颜色、字体、间距、明暗主题和响应式规则 |
| `components/layout/`          | 顶栏、导航、移动菜单和页脚             |
| `components/page/`            | 通用页面标题                           |
| `components/ui/site-link.tsx` | 静态导出模式下的站内链接约束           |
| `features/theme/`             | 主题状态和 HeroUI 主题按钮             |
| `features/locale/`            | HeroUI 语言菜单与当前路径切换          |
| `features/home/`              | 原型首页首屏和近期文章结构             |
| `site.config.ts`              | 品牌、作者、头像和社交链接             |
| `lib/i18n.ts`                 | 语言判断、路径转换和公共文案           |
| `lib/navigation.ts`           | 导航配置与选中状态计算                 |

### 关键决策

1. 原型截图和 `prototype/styles.css` 是当前视觉基准。主背景保持纯白，深色模式使用中性黑灰，主色保留青绿色。
2. Noto Sans SC 与 Noto Serif SC 通过 Fontsource 本地打包，页面不依赖 Google Fonts 在线请求。
3. 主题使用 `next-themes` 管理，并把结果写到根元素的 `data-theme`。主题按钮、移动菜单和卡片使用 HeroUI 3。
4. 顶栏是小范围 Client Component，用于读取当前路径、控制移动菜单和更新文档语言。正文与首页卡片保持 Server Component。
5. 中英文使用两套静态路由。语言切换使用只有图标的 HeroUI Select，弹层列出完整语言名称，并保留当前页面位置，例如 `/books/` 与 `/en/books/` 相互切换。
6. 搜索按钮在搜索模块完成前不显示，避免留下无响应入口。
7. 所有导航目标都提供真实页面。尚未进入内容阶段的归档、图书和关于页面先展示通用标题结构。
8. 字号在各响应式断点使用固定值，避免随视口连续缩放造成不可预测的换行。
9. 站内链接关闭 Next.js RSC 自动预取。Next.js 16 的分段预取地址与静态导出目录不一致，会在纯静态服务器产生 404；页面点击导航仍使用 Next.js 客户端路由。

## 实现细节

### 主题

`ThemeProvider` 在浏览器加载前写入主题初始化脚本，减少明暗主题闪烁。`ThemeToggle` 同时渲染太阳和月亮图标，由 `data-theme` 控制显示，服务端和客户端不需要根据浏览器主题渲染两套不同结构。

### 语言与导航

`getLocaleFromPath` 通过 `/en` 前缀判断语言。`getLocalizedPath` 先移除原语言前缀，再按目标语言重新组装路径。导航高亮也在移除语言前缀后计算，因此中英文页面共用同一套规则。

语言触发器不显示当前语言缩写和下拉箭头，减少顶栏宽度和视觉噪音。HeroUI Select 负责键盘操作、焦点、选中状态和弹层定位；选择语言后继续调用现有 `getLocaleSwitchPath`，内容译文和无译文回退规则保持不变。

路径转换属于纯函数，单元测试覆盖首页、内容页、前缀移除和子页面高亮。浏览器测试覆盖真实语言切换和移动菜单跳转。

### HeroUI 使用边界

- `Button` 负责主题和移动菜单等真实命令。
- `Select` 与 `ListBox` 负责语言切换菜单和选中状态。
- `Tooltip` 为主题按钮提供文字说明。
- `Card` 负责作者、主题和文章这些重复内容块。
- 页面路由继续使用 Next.js `Link`，保证静态页面之间可以正常导航。

项目通过 HeroUI 子路径导入组件，减少首页加载未使用的组件代码。

### 响应式布局

- 大于 1020px 时显示完整桌面导航，首页使用正文与侧栏两列布局。
- 760px 到 1020px 使用移动菜单，同时保留首页两列布局。
- 小于等于 760px 时首页改为单列，作者卡和主题卡接在主要操作下方，近期文章改为纵向排列。

### 素材

头像路径和替代文字由 `site.config.ts` 提供。首页文章图来自内容数据。图片带固定尺寸或稳定容器，避免加载时改变页面布局。

## 当前限制

- 公共配置当前支持一位作者和两种语言。
- 新增社交平台图标仍需要在首页作者卡补充对应图标映射。
