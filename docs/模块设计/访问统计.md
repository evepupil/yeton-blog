## 设计

### 职责

访问统计模块负责在页面空闲后加载 Umami Cloud 脚本，并提供旧站公开统计面板入口。它只处理匿名访问统计，不读取文章内容，不设置 Cookie，也不影响页面渲染、搜索或评论。

### 目录结构

| 路径                                           | 用途                                       |
| ---------------------------------------------- | ------------------------------------------ |
| `site.config.ts`                               | Umami 服务地址、Website ID、分享 ID 与开关 |
| `lib/analytics/config.ts`                      | 校验公开配置并生成脚本和分享地址           |
| `components/integrations/analytics-script.tsx` | 延后加载 Umami 脚本                        |
| `components/layout/site-document.tsx`          | 在中英文公共文档中挂载统计脚本             |
| `tests/unit/analytics-config.test.ts`          | 配置开关、地址和 ID 校验单测               |

### 关键决策

1. 继续使用旧站的 Umami Cloud Website ID，保留已有历史统计和公开面板。
2. 使用 Next.js `lazyOnload` 策略，浏览器完成页面加载并进入空闲阶段后再请求统计脚本。
3. Umami 加载失败时不显示全页错误，也不阻塞任何博客功能；公开统计链接仍可单独打开。
4. 配置启用时要求 HTTPS 服务地址、UUID 格式的 Website ID 和安全字符组成的分享 ID，错误配置会阻止构建。
5. CSP 只额外放行 `https://cloud.umami.is` 的脚本和请求，不放宽 frame、表单或其他来源。

## 实现细节

`resolveUmamiConfig` 是统计配置的唯一入口。开关关闭时返回 `null`；开启时校验配置并生成 `/script.js` 与 `/share/<id>` 的完整 HTTPS 地址。`AnalyticsScript` 只接收解析后的公开值，使用 `data-website-id` 交给 Umami 识别站点。

首页和页脚只有在配置有效且存在分享 ID 时显示统计链接。统计服务不可用不会触发 React 状态变化，正文和导航仍按纯静态页面工作。

## 测试方法

- 单测覆盖启用、关闭、HTTPS 限制、Website ID 和分享 ID。
- `pnpm build` 检查 Cloudflare `_headers` 已放行 Umami 域名，并检查静态页面包含 Website ID。
- E2E 拦截真实 Umami 请求，避免测试产生访问数据，同时确认页面无控制台错误。

## 当前限制

- 当前只做站点和页面访问统计，没有在页面内展示单篇文章浏览数。
- Umami Cloud 的统计字段和数据保留策略由 Umami 服务控制；更高的数据控制要求需要改为自托管 Umami。
