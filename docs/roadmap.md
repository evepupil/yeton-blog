# HeroUI Blog 实现路线图

## 1. 目标与交付边界

本项目将现有 HTML 原型实现为可长期维护的个人博客，并部署到 Cloudflare Pages。

首个正式版本需要完成：

- 还原原型中的首页、文章列表、文章详情、标签、归档和图书页面
- 支持 Markdown/MDX 内容、中文/英文内容和构建期全文搜索
- 完成 SEO、RSS、站点地图、404、主题切换和移动端适配
- 建立格式化、静态分析、单元测试、端到端测试和构建门禁
- 通过 Cloudflare Pages 自动构建和发布

首个版本上线后再接入评论、Notion 同步、访问统计和 AI 搜索。这些功能不会阻塞博客基础版本发布。

## 2. 已确定的技术方案

| 领域     | 方案                       | 说明                                                             |
| -------- | -------------------------- | ---------------------------------------------------------------- |
| Web 框架 | Next.js App Router         | 页面默认使用 Server Component，仅把交互区域放入 Client Component |
| 开发语言 | TypeScript 严格模式        | 开启 `strict`，内容数据和接口输入都做类型校验                    |
| 组件库   | HeroUI                     | 用于导航、按钮、卡片、标签、弹窗、输入框、抽屉和进度条           |
| 样式     | Tailwind CSS + 设计变量    | 颜色、字号、间距、圆角和明暗主题集中管理                         |
| 内容源   | 仓库内 Markdown/MDX        | frontmatter 使用 Zod 校验，构建时生成页面和索引                  |
| 搜索     | 构建期索引 + MiniSearch    | 搜索逻辑在浏览器运行，搜索模块按需加载                           |
| 部署     | Cloudflare Pages 静态导出  | Next.js 使用 `output: "export"`，发布目录为 `out`                |
| 动态接口 | Cloudflare Pages Functions | AI 等动态能力通过 `/api/*` 提供，和静态站点分开维护              |
| 评论     | Giscus                     | 放在增强阶段，不保存自建用户数据                                 |
| 包管理   | pnpm                       | 锁定依赖版本，开发机和 CI 使用同一份 lockfile                    |

### 静态部署约束

- 所有文章、标签、归档和图书详情都必须在构建时确定路径。
- 图片保存到仓库并预先处理尺寸；使用 `next/image` 时启用静态导出兼容配置。
- App Router 内不放依赖 Node.js 服务端运行时的接口。
- Notion 内容在构建前同步到本地，网页访问时不直接请求 Notion。
- Pages Functions 只承载确实需要在线执行的功能，例如 AI 流式回答。

## 3. 页面与路由

| 路由            | 功能                                   | 首版         |
| --------------- | -------------------------------------- | ------------ |
| `/`             | 中文首页、作者资料、标签和近期文章     | 是           |
| `/posts`        | 文章列表、标签筛选和分页               | 是           |
| `/posts/[slug]` | 文章正文、目录、上一篇/下一篇          | 是           |
| `/tags/[tag]`   | 指定标签下的文章                       | 是           |
| `/archives`     | 按年份归档                             | 是           |
| `/books`        | 图书列表和阅读进度                     | 是           |
| `/books/[slug]` | 图书详情和章节目录                     | 是           |
| `/links`        | 友链                                   | 否，增强阶段 |
| `/en/*`         | 对应英文页面                           | 是           |
| `404.html`      | 由 App Router `not-found` 页面静态导出 | 是           |
| `/rss.xml`      | 中文 RSS                               | 是           |
| `/en/rss.xml`   | 英文 RSS                               | 是           |

站内链接都在当前标签页导航。外部社交链接、友链和来源链接按内容配置决定是否打开新标签页。

## 4. 目标目录结构

```text
app/
  (zh)/                 # 默认中文路由
  (en)/en/              # 独立英文根布局与英文路由
  globals.css
  sitemap.ts
  robots.ts
components/
  ui/                   # 对 HeroUI 的轻量封装
  layout/               # 顶栏、页脚、移动导航
features/
  home/                 # 首页组合组件
  posts/                # 列表、卡片、筛选、正文和目录
  search/               # 搜索弹窗与索引加载
  archives/             # 年份归档
  books/                # 图书和阅读进度
  theme/                # 主题状态与切换
  locale/               # 语言配置与链接生成
content/
  posts/zh/
  posts/en/
  books/zh/
  books/en/
lib/
  content/              # 读取、校验、查询、排序和索引生成
  seo/                  # 元数据、RSS 和站点地图
  cloudflare/           # Pages Functions 客户端契约
functions/
  api/                   # Cloudflare Pages Functions
scripts/                # 内容检查、Notion 同步和图片处理
tests/
  unit/
  e2e/
docs/
  模块设计/
```

页面只负责组织模块。内容查询、排序、字数、阅读时间和目录提取等规则放在 `lib/content`，这样可以单独测试，也方便后续替换内容来源。

## 6. 项目门禁

阶段 0 完成后，提交前统一执行：

```bash
pnpm format:check
pnpm lint
pnpm typecheck
pnpm test
pnpm test:e2e
pnpm build
git diff --check
```

门禁规则：

- 任何命令失败都不能进入部署流程。
- 核心内容规则和状态计算必须有单元测试。
- 主题、配色、布局和组件外观通过浏览器检查与截图验证。
- E2E 只覆盖真实用户流程，避免重复验证组件内部实现。
- 第三方服务统一覆盖正常、超时、限流和不可用状态。

## 7. 性能与可访问性基线

- 首页正文和文章摘要由服务端组件生成，避免把整份内容数据发到浏览器。
- 搜索、评论、统计和 AI 面板按需加载。
- 独立内容读取并行执行，避免构建和请求过程出现无意义的串行等待。
- 长列表启用分页或分段渲染，图片固定宽高，避免页面加载时跳动。
- 所有交互控件支持键盘操作、可见焦点和屏幕阅读器名称。
- 动画遵守 `prefers-reduced-motion`，正文颜色满足 WCAG AA 对比度。

首版发布前记录 Lighthouse 基线，目标如下：

| 项目             | 目标  |
| ---------------- | ----- |
| Performance      | >= 90 |
| Accessibility    | >= 95 |
| Best Practices   | >= 95 |
| SEO              | >= 95 |
| 页面布局偏移 CLS | < 0.1 |
